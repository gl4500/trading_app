LAYER 4 — INDIVIDUAL AGENTS (all run in parallel each cycle)

TechAgent

Rule-based technical analysis

  • RSI (14) — oversold/overbought
  • MACD (12/26/9) — cross + histogram
  • Bollinger Bands (20, 2σ)
  • Stochastic %K/%D
  • OBV divergence
  • Volume spike confirmation

BUY score ≥ 0.55

SELL score ≥ 0.55 + has position

Sells entire position

MomentumAgent

Price momentum + trailing stop

  • mom_5d ×5 + mom_10d ×3 + mom_20d ×2
  • Trend consistency score
  • Acceleration factor
  • Volume-weighted momentum

BUY score ≥ 0.30 + vw_mom > 0

SELL score ≥ 0.30 or mom < −2%

Trailing stop: drawdown > 3%

MeanReversionAgent

Z-score mean reversion (20-day)

  • z = (price − mean) / std
  • MR_BUY_ZSCORE = −1.5
  • MR_SELL_ZSCORE = +1.5
  • Early exit: −0.3 ≤ z ≤ 0.5 if P&L > 1%

BUY z ≤ −1.5

SELL z ≥ +1.5 or mean-reversion exit

conf = min(0.95, |z| / 3)

SentimentAgent (Ollama / GPT-4o-mini)

LLM news sentiment, throttled 90 min, batch ≤ 5 symbols

  • Input: 1D/5D/20D returns, vol, high/low dist, news headlines
  • LLM returns: sentiment + confidence + strength
  • adjusted_conf = confidence × {strong:1.0, moderate:0.8, weak:0.6}

BUY bullish + adjusted_conf ≥ 0.45

SELL bearish + adjusted_conf ≥ 0.45

HistoricalTrendsAgent

Seasonal + channel + multi-period momentum

  • composite = seasonal×0.20 + channel×0.30 + momentum×0.40 + volume×0.10
  • Seasonal: month-of-year bias + quarter effect
  • Channel: price in 5-yr high-low range (SMA-20 slope adjusted)
  • Momentum: weighted RoC 5/10/20/40d — alignment bonus ±0.20

BUY composite > +0.25, no position

SELL composite < −0.25, has position

ClaudeAgent (Claude Opus 4.6 / Ollama llama3.1:8b)

Full-context LLM — every 3rd cycle, ≤ 2 API calls/hr

  • Input: all indicators + composite signal + news + overnight catalysts + portfolio state + learning history + Gemini market view + risk assessor flags
  • STRONG BUY: catalyst + RSI < 65 + MACD+ + price > SMA20
  • STRONG SELL: negative + RSI > 65 + MACD− + price < SMA20
  • CONFLICTED: use technicals for timing, news for direction

Phase 1 Ollama Learning (OLLAMA_ONLY_MODE=1):

  • Top 5 profitable Claude trades prepended as few-shot EXAMPLES (reasoning → outcome)
  • Top 3 losses shown as AVOID PATTERN blocks
  • llama3.1:8b mimics Claude Opus decision style

Output: per-symbol {action, shares, confidence, reasoning}

Replays last decisions on non-API cycles

LAYER 5 — ENSEMBLE VOTING (EnsembleAgent, every cycle)

Weighted Vote — 6 agents (GeminiAgent excluded)
AgentBase Weight
ClaudeAgent29%
TechAgent23%
SentimentAgent17%
MomentumAgent14%
MeanReversionAgent9%
HistoricalTrendsAgent8%

buy_score = Σ(weight × conf) for BUY signals / total_weight

sell_score = Σ(weight × conf) for SELL signals / total_weight


Every 5 cycles — adaptive weight recalculation:

  • perf_score = 0.5 × win_rate + 0.5 × norm_sharpe
  • norm_sharpe = (sharpe + 2) / 5 [clamped −2 to +3]
  • raw = base × max(0.30, perf_score) × regime_multiplier
  • final = raw / Σ(all raw) [renormalised]

Regime detection (SPY/QQQ, every 5 cycles):

SignalCondition
A|SMA20 slope| > 0.4% per price
B>60% of last 10 days in slope direction
C5-bar volume > 20-bar avg × 1.10

ATR > 2.5% → VOLATILE (checked first)
2+ of A/B/C → TRENDING
otherwise → RANGING

Regime Multipliers
AgentTRENDINGRANGINGVOLATILE
MomentumAgent×1.50×0.55×0.65
TechAgent×1.20×1.10×0.80
HistoricalTrendsAgent×1.20×1.30×0.80
ClaudeAgent×1.10×1.00×1.40
SentimentAgent×0.80×1.20×1.20
MeanReversionAgent×0.40×1.60×0.60
Final Vote Decision

BUY buy_score ≥ 0.60 AND no existing position AND buy_score ≥ sell_score × 2.0 (margin of safety)

SELL sell_score ≥ 0.60 AND holding position

HOLD neither threshold met, or margin of safety fails

Position sizing:

  • target = portfolio_value × 15% × buy_score
  • target = min(target, available_cash × 0.95)
  • shares = floor(target / price × 100) / 100

LAYER 6 — RISK MANAGER GATE (before every trade)

RiskManager — any FAIL blocks the trade
  • Daily loss limit not breached (default 5% of portfolio)
  • Position size ≤ MAX_POSITION_SIZE per symbol (default 15%)
  • Sufficient cash available for trade value
  • No churn cooloff active (30 min after selling a symbol)
  • Sector concentration ≤ 35% of portfolio value
  • No short selling (long-only)

✓ PASS → Trade executed (Alpaca paper API)

✗ FAIL → Signal dropped (reason logged)

LAYER 7 — SCANNER AGENT (independent, every 30 min)

ScannerAgent — Agentic tool-use loop (Claude Opus + Ollama fallback)

Input: ~299 S&P 500 stocks (stock_universe.py)

Pre-screen: price change + volume ratio filter → top candidates

Tool-use loop: fetch bars, check technicals, assess news, score opportunity

Output: ranked BUY / SELL / WATCH + confidence + price target + catalyst

Persisted: data/scan_cache.json (survives restarts, is_stale flag)

Retry: up to 3 attempts with 5s/10s backoff on VRAM crash

ScannerPortfolioAgent (every cycle):

  • Reads cached scanner results
  • BUY conf ≥ 0.60, fresh scan only (no churn)
  • SELL explicit SELL rec from scanner
  • SELL not in BUY set + position loss ≥ 5%
  • Same RiskManager gate applies

Scanner symbols also injected into all agents' market_context each cycle

FULL CYCLE SUMMARY — Every 60 seconds (9:30–16:00 ET)

  1. Fetch market data — Alpaca OHLCV + Stooq long-term bars → build market_context for all watchlist symbols
  2. (Every 3rd cycle) GeminiAgent pre-pass → market_analysis injected into context as __gemini_market_view__
  3. 6 agents run in parallel → each returns BUY / SELL / HOLD per symbol
    (TechAgent · MomentumAgent · MeanReversionAgent · SentimentAgent · HistoricalTrendsAgent · ClaudeAgent)
  4. EnsembleAgent collects all signals → weighted vote per symbol
        → Every 5 cycles: recalculate regime + adaptive weights
        → BUY gate: threshold 0.60 AND margin-of-safety 2× vs sell signal
  5. RiskManager gate — checks daily loss / position size / cash / churn / sector limits
  6. Passing signals → Alpaca paper trade execution
  7. (Every 30 min) ScannerAgent runs independently → scan_cache.json updated
  8. ScannerPortfolioAgent reads cache → its own BUY / SELL signals → same RiskManager gate → Alpaca execution
  9. (24/7 background) Sentinel polls news every 5–15 min → catalysts queued → triggers scanner if score ≥ 2
