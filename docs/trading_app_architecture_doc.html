1. System Overview

The AI Trading App is a paper-trading competition engine that runs 9 autonomous agents concurrently. Each cycle (~60 seconds) the system fetches multi-source market data, builds a rich market_context dictionary, and dispatches it to every agent. Agents emit BUY / SELL / HOLD signals which are executed against each agent's independent virtual portfolio. The EnsembleAgent aggregates votes from 7 sub-agents using adaptive performance-weighted voting to produce the highest-quality trades.

Key invariants:
  • market_context[symbol] is always a dict — never iterate without isinstance(ctx, dict)
  • market_context["__overnight_catalysts__"] is a list, not a dict
  • Portfolio has no get_position() — access via portfolio.positions[sym].shares
  • NYSE hours only: 9:30–16:00 ET. No pre/after-hours trades executed.

3. Composite Signal — 5-Source Aggregator

data/signal_aggregator.py runs concurrently for each watchlist symbol before market_context is assembled. Each source emits a score in [-1, +1]. The weighted average becomes composite_signal.score.

4. The 9 Trading Agents

All agents inherit from BaseAgent. Each runs analyze(market_context, prices) → Dict[str, Signal]. The trading loop calls them concurrently via asyncio.gather, then persists new trades via save_trade().

5.2 Gemini → Claude Linkage

GeminiAgent is a pure context provider, not a voter.
Each cycle, GeminiAgent.get_market_view(market_context, watchlist) is called and returns a short narrative paragraph (e.g. "Tech sector showing momentum, rising rates a headwind for growth"). This is injected as market_context["__gemini_market_view__"] and appended to ClaudeAgent's prompt under "## Gemini Market View". Claude reads this as peer analysis and weighs it against its own assessment. Gemini's signals are not passed to EnsembleAgent.

5.3 Agent Signals → CNN Linkage (cross-cycle)

CNNReasoningAgent uses other agents' signals as a feature.
After each cycle, the trading loop collects all per-symbol signals from every agent and stores them in market_context["__agent_signals__"] = {sym: {agent_name: (action, confidence)}}. On the next cycle, CNNReasoningAgent reads this dict and prepends an "## Other Agent Signals This Cycle" section to its Ollama prompt. This lets the CNN agent use the ensemble's collective intelligence as a soft prior. This is a one-cycle lag — CNNAgent always reads last cycle's signals.

5.4 Scanner → All Agents Linkage

The scanner expands the universe of symbols all agents can trade.
When a fresh scan is available, the trading loop fetches full market context for scanner-recommended symbols and merges them into market_context (deduped against the watchlist). This means every agent — not just ScannerPortfolioAgent — can trade scanner picks. ScannerPortfolioAgent specifically acts on scanner BUY/SELL recommendations directly, while other agents get the full price/technical/news data for the same symbols.

5.5 Agent Performance → Ensemble Weights (adaptive feedback)

EnsembleAgent dynamically reweights based on live performance.
Every 5 cycles, _update_weights() reads each component agent's portfolio metrics (win_rate, sharpe_ratio), computes a performance score = 0.5×win_rate + 0.5×normalised_sharpe, and scales the base weight by that score. A floor of 30% of base weight prevents any agent from being zeroed out. Regime multipliers then layer on top (e.g. Momentum ×1.50 in trending, MeanRev ×1.60 in ranging).

6.3 Vote Decision Rule

For each symbol:
buy_score = Σ(weight × confidence) for BUY signals
sell_score = Σ(weight × confidence) for SELL signals

BUY if: buy_score ≥ ENSEMBLE_THRESHOLD (0.60) AND buy_score ≥ sell_score × MARGIN_OF_SAFETY (2.0)
SELL if: symmetric — sell_score ≥ 0.60 AND sell_score ≥ buy_score × 2.0
HOLD otherwise.

Setting OLLAMA_ONLY_MODE=1 routes all three LLM agents through local Ollama (no cloud calls, zero cost):

CNNReasoningAgent–Always OllamaOLLAMA_MODEL

RTX 2060 constraint: 6 GB VRAM fits only one Q4 model at a time. Set RESEARCH_MODEL=OLLAMA_MODEL to share the single loaded model across all agents. Never configure two different models simultaneously.

JSON enforcement: Ollama models often return markdown instead of raw JSON. The parser strips ```json``` fences and retries. System prompt includes: "CRITICAL: Output raw JSON only. The first character must be '{' and the last must be '}'."
